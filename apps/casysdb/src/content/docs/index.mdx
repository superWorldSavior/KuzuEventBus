## About CasysDB

CasysDB is an embedded graph database written in Rust. It links directly into your application process, ships as a single native library, and exposes bindings for Python (TypeScript bindings are in development). The engine speaks the ISO Graph Query Language (GQL) and relies on MVCC with snapshot isolation to provide consistent reads, Git-like branching, and point-in-time recovery.

Key properties:

- **Embedded runtime** — no external server to deploy or operate.
- **ISO GQL support** — standards-based query surface with familiar `MATCH / WHERE / RETURN` semantics.
- **MVCC transactions** — consistent snapshots, safe concurrent reads, point-in-time recovery.
- **Branching workflow** — create isolated branches for experiments, apply PITR to recover known good states.
- **Python SDK** — native bindings with an idiomatic session/ORM layer; TypeScript SDK in progress.

```python
from casys_db import Database

db = Database(":memory:")
branch = db.default_branch()

branch.query("""
  CREATE (alice:Person {name: 'Alice', age: 30})
  RETURN alice
""")

rows = branch.query("""
  MATCH (p:Person)
  WHERE p.age >= 18
  RETURN p.name
""")
```
